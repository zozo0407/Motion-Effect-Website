# Wrapped Parts UI Config (V2)

## Goal

When `AI_V2_OUTPUT_MODE=wrapped_parts`, the model currently returns two code snippets (`setup` / `animate`) which are wrapped into a full `EngineEffect`. The Lab left panel UI is driven by `EngineEffect.getUIConfig()`. We want **effect-specific controls** generated by the model, while keeping the pipeline stable and safe.

## Scope

- Applies only to `/api/generate-effect-v2` when `AI_V2_OUTPUT_MODE=wrapped_parts`.
- UI is sent via the existing `HANDSHAKE -> UI_CONFIG` message flow in [templates.js](file:///Users/bytedance/Downloads/cupcut-website/src/site/templates.js).
- Control types allowed: `color`, `range`, `checkbox`, `select`.
- Max controls: **6**.

Non-goals:
- No new UI widgets in Lab.
- No file upload / buttons / textareas in model-generated UI for now.

## Output Protocol (Strict)

The model must output **three** sections:

1. `setup` snippet (function body only)
2. `---SPLIT---`
3. `animate` snippet (function body only)
4. `---UI---`
5. **Pure JSON array** (no markdown, no comments, no extra text)

Example:

```text
// === setup ===
this.mesh = ...
---SPLIT---
// === animate ===
this.mesh.rotation.y = time * 0.3;
---UI---
[
  { "bind": "primaryColor", "name": "主色调", "type": "color", "value": "#00f2ff" },
  { "bind": "speed", "name": "速度", "type": "range", "min": 0.1, "max": 5, "step": 0.05, "value": 1.0 }
]
```

## Server Prompt Rules

In [server.js](file:///Users/bytedance/Downloads/cupcut-website/server.js) wrapped_parts system prompt:

- Explicitly document `---UI---` section and the exact JSON schema.
- Require `bind` keys to be short identifiers: `^[a-zA-Z_][a-zA-Z0-9_]{0,31}$`.
- Require at most 6 items in the UI array.
- For `select`, require `options` as string array (max 10).
- Forbid any non-JSON output after `---UI---`.

## Parsing & Validation

Implemented in [v2-wrapped-parts.cjs](file:///Users/bytedance/Downloads/cupcut-website/tools/creator/v2-wrapped-parts.cjs):

- Extend `parseAIOutput(rawText)` to:
  - Strip code fences and `<think>/<analysis>` prologues (existing behavior).
  - Split by `---UI---` (optional).
  - Parse JSON if present; if parse fails, treat as `uiConfig = null`.
- Validate UI config:
  - Must be an array.
  - Each item must include: `bind`, `type`, `name`, `value` (except `checkbox` where value can default to `false`).
  - Allowed `type`: `color|range|checkbox|select`.
  - Enforce `bind` regex + length.
  - Clamp numeric fields (`min/max/step/value`) into sane bounds.
  - Max items: 6. Excess items are dropped.
  - `select.options` max length: 10; must be strings.
  - Invalid items are dropped, not fatal.

## Wrapping Behavior

Update `wrapAsEngineEffect(setup, animate, uiConfig)`:

- `getUIConfig()`:
  - If `uiConfig` exists and passes validation -> return it.
  - Else -> return a safe default config (color/intensity/speed/scale) so Lab always has something.
- `setParam(key, value)`:
  - Always writes into `this.params`.
  - Best-effort to propagate:
    - `this.material.uniforms['uXxx']` and `this.material.color`.
    - `this.group.scale.setScalar()` for `scale`.
- `onUpdate(ctx)`:
  - Multiply `time/deltaTime` by `this.params.speed` to make speed control meaningful in most effects.

## Failure/Degrade Strategy

- If UI JSON is missing or invalid: do NOT fail generation; fall back to default UI config.
- If `setup` snippet is missing/empty: fail generation (existing behavior).

## Acceptance Checks

- For wrapped_parts output, the generated code includes `getUIConfig()` returning:
  - Model-provided UI when valid and <= 6 items.
  - Default UI when missing/invalid.
- Lab left panel shows controls after opening Lab (HANDSHAKE).
- Changing a control sends `UPDATE_PARAM` and is observable in effect for common patterns.
