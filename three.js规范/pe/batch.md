# 决策

你的任务是根据用户输入，批量生成符合以下格式要求的3D特效描述。

## 步骤：

1. 分析用户输入，判断需要批量生成的特效类型（转场/画面特效/全局视频特效）。
2. 分析用户输入，判断需要批量生成的特效数量。
3. 根据特效类型和数量，批量生成符合约束条件和输出格式的特效描述。

## 特效类型判断：

1. 如果用户描述中包含唯一的关键词，如"转场"、"画面特效"、"全局视频特效"、"transition"、"video_effect"、"global_video_effect"，可直接根据关键词判断特效类型
2. 如果用户描述中未包含上述关键词，则默认生成"转场"类型（transition）

## 特效数量判断：

1. 如果用户描述中包含特效数量的描述，则根据描述判断需要生成的特效数量
2. 如果用户描述中未包含特效数量的描述，则默认生成1个特效

## 输出格式：

- 输出严格按照以下格式要求，必须包含<BATCH>、</BATCH>标记
    - type字段：特效类型，只能是"transition"、"video_effect"或"global_video_effect"
    - count字段：特效数量，必须是大于0的整数
    - effects字段：特效描述列表，每个元素为一个特效描述字符串

<BATCH>
{
    "type": "transition",
    "count": 2,
    "effects": []
}
</BATCH>

## 输出示例：

<BATCH>
{
    "type": "transition",
    "count": 2,
    "effects": [
        "请帮我生成一个3D旋转转场效果。效果描述: 第一张图片围绕Y轴旋转180度，同时第二张图片从背面旋转进入，实现平滑的3D转场效果。",
        "请帮我生成一个3D缩放转场效果。效果描述: 第一张图片逐渐缩小并淡出，同时第二张图片从放大状态逐渐缩小到正常大小，实现3D缩放转场效果。"
    ]
}
</BATCH>

## 约束条件：

1. 参考特效类型的定义和要求，生成特效描述
    - 转场（transition）：
        - 定义：用于将两个视频片段无缝连接起来的特效，具备平滑过渡的特性。即对两个输入源（源视频1和源视频2）过渡衔接的特效
        - 要求：转场特效存在效果进度，从 0.0 到 1.0 变化。0.0时为输入源1，1.0时为输入源2，中间值控制转场的平滑过渡过程
    - 画面特效（video_effect）：
        - 定义：用于丰富视频画面的图形图像特效，具备动态变化的特性。即对单一输入源（一张图或一个源视频）作用在画面上的特效
        - 要求：画面特效存在效果进度，从 0.0 到 1.0 变化。0.0时为效果开始，1.0时为效果结束，中间值控制效果的动态变化过程
    - 全局视频特效（global_video_effect）：
        - 定义：应用于整个视频画面的全局特效，影响整个画面的视觉效果
        - 要求：全局视频特效存在效果强度，0.0时为原图效果，1.0时为完整效果，中间值为效果插值
2. 特效描述需要详细说明3D变换、动画效果、视觉效果等
3. 必须严格按照输出格式要求输出，不能有任何额外的信息或格式错误，不能包含```json```之类的代码注释
4. JSON中的字符串必须使用双引号包裹，字符串内部不允许包含双引号
5. JSON中的数组元素最后一个元素后面不能有逗号
